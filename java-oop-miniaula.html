O que é Orientação a Objetos?

POO é uma forma de organizar e pensar o código como o mundo real funciona — com coisas que têm características e comportamentos.

No mundo real
🚗
Um carro tem características: cor, modelo, potência.
Um carro tem comportamentos: ligar, acelerar, frear.
💡 O que é POO então?
POO é o paradigma de programação onde você modela o código como objetos do mundo real — cada objeto tem seus próprios dados e suas próprias ações. Em vez de escrever código solto, você organiza tudo dentro de objetos que se comunicam entre si.

Por que usar POO?

Antes de aprender como funciona, entenda o porquê de usar Orientação a Objetos.

🤔 O problema sem POO
Sem POO, o código vira uma bagunça difícil de entender e reutilizar. Com POO, cada coisa fica no seu lugar.
✅ Vantagens
Organização — código limpo e separado.
Reuso — escreva uma vez, use várias.
Manutenção — fácil de corrigir e evoluir.
🏭

Pense numa fábrica de carros

Sem organização, cada operário faz tudo de um jeito diferente — vira caos. Com POO é como ter um manual padronizado: todo mundo sabe onde está cada peça e como montar.

🌍 No mercado de trabalho
Java, Python, C#, Kotlin — as linguagens mais usadas nas empresas são orientadas a objetos. Aprender POO é aprender a linguagem do mercado.

Classe: o molde

Uma Classe é como uma receita ou planta baixa. Ela descreve como um objeto será — mas ainda não é o objeto em si.

Carro.java 
// Classe = Molde / Receita
public class Carro {

    // Atributos (características)
    String cor;
    String marca;
    int velocidade;

    // Método (ação)
    void acelerar() {
        velocidade += 10;
        System.out.println("Vrum! Vel: " + velocidade);
    }
}
Atributos
Características do objeto.
Ex: cor, marca, velocidade
Métodos
Ações que o objeto faz.
Ex: acelerar(), frear()

Objeto: a instância real

Se a Classe é o molde, o Objeto é a peça que saiu do molde. Cada objeto tem seus próprios valores!

Main.java 
public class Main {
    public static void main(String[] args) {

        // Criando objetos com construtor
        Carro fusca   = new Carro("azul",     "VW",      50);
        Carro ferrari = new Carro("vermelho", "Ferrari", 500);

        fusca.acelerar();   // age só sobre o fusca
        ferrari.acelerar(); // age só sobre a ferrari
    }
}
🔑 Palavra-chave: new
Usamos new para criar um objeto a partir de uma classe. Cada objeto vive de forma independente na memória.
🔧 O que é o Construtor?
O construtor é um método especial da classe que roda automaticamente no momento do new. Ele serve para inicializar o objeto já com seus valores — em vez de atribuir cada atributo separadamente depois.

Os 4 Pilares da POO

Toda linguagem POO — incluindo Java — é baseada nestes quatro conceitos fundamentais.

📦

Encapsulamento

Esconder detalhes internos. Só mostrar o necessário.

🧬

Herança

Uma classe pode herdar atributos e métodos de outra.

🎭

Polimorfismo

O mesmo método se comporta diferente em cada classe.

🔷

Abstração

Focar no que importa, ignorando detalhes desnecessários.

🛍️ Pense assim na prática
Pense numa sacola de supermercado 🛍️ — dentro dela você coloca tudo que precisa. Ela aceita qualquer coisa. Mas na hora de dar algo ao seu filho, você não entrega a sacola inteira: você tira apenas o que ele pode receber.
